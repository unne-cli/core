# Поддержка прокси

Unne CLI может маршрутизировать подключение к серверу через HTTP или SOCKS5 прокси.

## Использование

### Флаг CLI

```bash
unne http 3000 --proxy socks5://127.0.0.1:1080
unne http 3000 --proxy http://user:pass@proxy.corp.com:8080
```

### Файл конфигурации

Глобальный прокси (для всех туннелей):

```yaml
proxy:
  url: "socks5://127.0.0.1:1080"
```

Прокси для отдельного туннеля:

```yaml
tunnels:
  - name: web
    protocol: http
    upstream: localhost:3000
    proxy:
      url: "http://corpproxy:8080"
```

## Поддерживаемые типы прокси

| Схема | Протокол | Аутентификация |
|-------|----------|----------------|
| `socks5://` | SOCKS5 | Имя пользователя/пароль через URL |
| `http://` | HTTP CONNECT | Basic-аутентификация через URL |
| `https://` | HTTPS CONNECT | Basic-аутентификация через URL |

## Аутентификация

Включите учетные данные в URL прокси:

```
socks5://user:password@proxy:1080
http://user:password@proxy:8080
```

## Серверные ограничения

Администратор сервера может отключить использование прокси для конкретного пользователя. Если у вашей учетной записи нет разрешения `can_use_proxy`, настройка прокси игнорируется на уровне подключения, но не вызывает ошибку.
