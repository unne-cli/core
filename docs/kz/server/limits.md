# Лимиттер мен квоталар

Unne Server ресурстарды пайдалануды бақылау үшін пайдаланушы бойынша лимиттерді қолдайды.

## Қолжетімді лимиттер

| Лимит | Сипаттама | Әдепкі |
|-------|-----------|--------|
| `max_tunnels` | Бір уақытта максимум туннельдер | `0` (шектеусіз) |
| `max_devices` | Бір уақытта максимум токен қосылыстары | `0` (шектеусіз) |
| `allowed_protocols` | Рұқсат етілген туннель протоколдары | `http,tcp` |
| `traffic_limit` | Кезең бойынша максимум трафик (байт) | `0` (шектеусіз) |
| `traffic_period` | Лимит кезеңі: `daily`, `monthly`, `quarterly` | `monthly` |
| `skip_warning` | Клиенттерге браузер ескертуін өткізіп жіберуге рұқсат | `false` |
| `can_use_proxy` | Прокси қосылыстарына рұқсат | `true` |

## Лимиттердің қолданылу тәртібі

Лимиттер клиент қосылған кездегі қол алысу кезінде тексеріледі:

1. **Токенді тексеру** -- токен бар және белсенді
2. **Пайдаланушыны тексеру** -- пайдаланушы бар және белсенді
3. **Протоколды тексеру** -- сұралған протокол `allowed_protocols` ішінде
4. **Құрылғы лимиті** -- осы пайдаланушының белсенді токендерінің саны
5. **Туннель лимиті** -- осы пайдаланушының белсенді туннельдерінің саны
6. **Трафик лимиті** -- ағымдағы кезеңдегі жинақталған байттар

Кез келген тексеру сәтсіз болса, қосылыс тиісті `STATUS_UNNE_*` қате кодымен қабылданбайды.

## Трафикті бақылау

Трафик әр туннель бойынша бақыланады және әр 30 секунд сайын деректер базасына жазылады. Есептегіштер кіріс және шығыс байттарды қамтиды.

### Трафик кезеңдері

| Кезең | Қайта орнату |
|-------|-------------|
| `daily` | Түн ортасы (сервер уақыт белдеуі) |
| `monthly` | Әр айдың 1-і |
| `quarterly` | Қаңтар 1, Сәуір 1, Шілде 1, Қазан 1 |

## Мысалдар

### Тегін тариф: тек HTTP, 3 туннель, айына 1ГБ

```
max_tunnels: 3
allowed_protocols: http
traffic_limit: 1073741824
traffic_period: monthly
max_devices: 1
```

### Про тариф: шектеусіз

```
max_tunnels: 0
allowed_protocols: http,tcp
traffic_limit: 0
max_devices: 0
skip_warning: true
```

## Лимиттерді орнату

Лимиттерді мына жолдармен орнатуға болады:
- **Әкімші панелі** -- пайдаланушыны өзгерту формасы
- **CLI** -- `unns user create` (негізгі), содан кейін толық лимиттер үшін әкімші панелі арқылы өзгерту
